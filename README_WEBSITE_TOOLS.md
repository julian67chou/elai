# 伊萊診所網站管理工具

本目錄包含伊萊診所網站的管理工具和標準作業程序（SOP），旨在規範網站更新流程，避免資源遺失和部署問題。

## 文件結構

```
elai-clinic/
├── WEBSITE_UPDATE_SOP.md          # 標準作業程序（完整版）
├── check_website_resources.sh     # 資源完整性檢查腳本
├── test_website_locally.sh        # 本地測試腳本
├── pre_deployment_checklist.sh    # 部署前檢查清單
├── deploy_elai_clinic.sh          # 標準部署腳本
├── verify_deployment.sh           # 部署驗證腳本
├── rollback_website.sh            # 快速回滾腳本
├── verify_rollback.sh             # 回滾驗證腳本
└── README_WEBSITE_TOOLS.md        # 本文件
```

## 快速開始

### 1. 資源檢查
在進行任何修改前，檢查網站資源是否完整：
```bash
./check_website_resources.sh
```

### 2. 本地測試
修改後進行本地測試：
```bash
./test_website_locally.sh
```

### 3. 部署前檢查
部署前運行完整檢查：
```bash
./pre_deployment_checklist.sh
```

### 4. 標準部署
通過檢查後進行部署：
```bash
./deploy_elai_clinic.sh "修改描述"
```

### 5. 部署驗證
部署後驗證網站狀態：
```bash
./verify_deployment.sh
```

### 6. 快速回滾（如有問題）
```bash
./rollback_website.sh [commit-hash]
```

查看完整 SOP: [WEBSITE_UPDATE_SOP.md](WEBSITE_UPDATE_SOP.md)

### 腳本功能詳解

#### 1. check_website_resources.sh
檢查網站所有資源檔案（圖片、CSS、JS）是否存在，避免資源遺失問題。

**檢查項目：**
- HTML 中的圖片引用
- CSS 中的背景圖片
- assets 目錄中的圖片檔案
- JavaScript 和 CSS 檔案

#### 2. test_website_locally.sh
啟動本地測試伺服器，進行全面功能測試。

**測試項目：**
- 網站可訪問性
- 關鍵內容檢查
- 圖片資源載入
- 內部連結檢查
- 表單元素檢查
- SEO 標籤檢查

#### 3. pre_deployment_checklist.sh
部署前的強制檢查清單，必須全部通過才能部署。

**檢查項目：**
- Git 狀態檢查
- 資源完整性檢查
- 本地功能測試
- 圖片檔案大小檢查
- HTML 語法檢查
- 外部連結檢查
- 響應式設計檢查
- 備份狀態檢查
- 檔案權限檢查
- Git 遠端設定檢查

#### 4. deploy_elai_clinic.sh
標準部署流程，自動化執行所有部署步驟。

**執行步驟：**
1. 部署前檢查
2. 創建部署前備份
3. 檢查 Git 狀態
4. 提交更改
5. 推送到 GitHub
6. 創建 Vercel 觸發文件
7. 創建部署報告
8. 創建驗證腳本

#### 5. verify_deployment.sh
驗證部署後的網站狀態。

**驗證項目：**
- 網站可訪問性
- 關鍵內容檢查
- 圖片載入檢查
- 更新時間檢查
- HTTP 狀態碼檢查

#### 6. rollback_website.sh
快速回滾到指定版本。

**功能：**
- 顯示最近提交記錄
- 驗證目標提交
- 創建回滾備份
- 執行本地回滾
- 可選強制推送到遠端
- 創建回滾報告
- 創建驗證腳本

## 基於歷史問題的預防措施

### 問題回顧
1. **圖片資源遺失**：提交 `0ce65e1` 中將 `yilan-landscape.jpg` 改為不存在的 `hero-image.jpg`
2. **院長照片遮擋文字**：圖片尺寸過大，影響版面佈局

### 預防措施
1. **強制資源檢查**：所有部署前必須通過資源完整性檢查
2. **圖片路徑雙重驗證**：修改圖片路徑時驗證目標檔案存在
3. **圖片大小限制**：檢查並警告過大圖片檔案
4. **完整備份系統**：每次部署前自動創建備份
5. **詳細部署報告**：記錄所有操作，方便追蹤和回滾

## 最佳實踐

### 1. 小步迭代
- 頻繁提交小更改
- 避免大規模一次性修改
- 每次修改都有明確目的

### 2. 完整測試
- 修改後立即本地測試
- 部署前運行完整檢查
- 部署後驗證網站狀態

### 3. 詳細記錄
- 清晰的提交訊息
- 完整的部署報告
- 問題分析和預防措施

### 4. 安全第一
- 重大修改前必須備份
- 有完整的回滾方案
- 緊急聯絡資訊齊全

## 緊急情況處理

### 網站無法訪問
1. 檢查 Vercel 部署狀態
2. 運行驗證腳本確認問題
3. 參考部署報告中的回滾方案
4. 從備份恢復或回滾到上個版本

### 圖片資源遺失
1. 從備份目錄恢復圖片
2. 更新 HTML 中的圖片路徑
3. 重新部署

### 樣式錯誤
1. 恢復 CSS 備份
2. 檢查瀏覽器緩存
3. 使用無痕模式驗證
